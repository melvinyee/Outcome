1.Explain the importance of codes readability.

Answer

The readability of the code is not actually for the compiler, the interpreter, but for the people. Code with good readability should be easy to understand, easy to maintain, easy to expand.Whenever you see a complex logic, a huge expression, a lot of variables, you should think about how to optimize them. Because these will increase your mind pressure, to know that each person's short-term memory is very limited. When you have to think too much about things, it is possible to generate bugs in unknowingly.

Do one thing at a time. At the same time doing a few things in the code will make it difficult to understand, a code block may be done initialization object, parse the input, in the interface display. Too much different business code entangled together will make it difficult for you to understand its logic clearly. Therefore, we should be able to organize the code as clear as possible, we can not only split a large function into a number of smaller functions to distinguish between boundaries, you can also organize the code within the function, making it feel like a separate logic segment.
